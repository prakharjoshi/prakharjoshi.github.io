Setting up Django

Setting up Production and Development Enviornment

Posted by Prakhar Joshi on June 6, 2014

Settings is one of the pathetic things in Django framework as it includes providing paths for almost all things like database, staticfiles,templates and other settings too. There are many ways to handle settings in django. When you start a project you get a settings.py file in your project name directory. But once you configured settings for production environment then you can't test your app locally. That sounds wiered. Right? So there is what I did to resolve this problem. Instead of having settings.py file(which is provided with the start of project) I have created a settings folder which contains base.py,local.py and production.py in it. You can see my project on link The basic Idea behind this is that we put the basic settings which we will gonna use in both local and production server in base.py and will import base.py in both local.py and production.py

BASE.PY :-

It will basically gonna include the settings of SECRET_KEY, PATH_CONFIGURATIONS, INSTALLED_APPS, MIDDLEWARE_CLASSES, EMAIL_SETTINGS and many more according to your project.

LOCAL.PY :-

It will first of all gonna import the base.py file so that we don't have to set settings for things which were included in base.py. Now we will configure our DATABASE here and DEBUG = TRUE for local settings. You can also add debug_toolbar in your installed app and debug_toolbar.middleware.DebugToolbarMiddleware in your middleware classes and if you want to activate your debug_toolbar in local enviornment you have to modify your main urls.py file by adding :- import debug_toolbar if settings.DEBUG: urlpatterns += patterns('', url(r'^__debug__/', include(debug_toolbar.urls)), ) This will gonna include a debug_toolbar for your local app running so that it will help you in debugging your errors.

PRODUCTION.PY:-

It will also first of all gonna import base.py. Then we will configure our DATABASE for our production environment and DEBUG = FALSE is compulsory. Then just follow the settings instruction from which you are deploying your app. IMPORTANT :- As you are making a settings folder instead of settings.py you must include __init__.py inside the settings folder so that it will show that there is some python stuff inside the folder. To run the app in local environment just run python manage.py runserver --settings=mysite.settings.local in the terminal. To run the app in production enviornment just run :- python manage.py runserver --settings=mysite.settings.production in the terminal. Hope this will gonna help you as its good to work with same settings for production and local. Cheers!!

Setting up Django

Setting up Production and Development Enviornment

Posted by Prakhar Joshi on June 6, 2014

Django has the most powerful and time saving inbuilt modules for authentication, E-mail sending, admin and a lot more. But there are always two sides of coin, On one hand it seems beneficial to have inbuilt modules but at same time they are quite vexed.

One of the inbuilt feature of django is mail sending feature. Sending email on request of user is one of the beloved feature of web development. Django provides an inbuilt setup for sending mails. We just have to import proper modules and we have to set server for sending mails.

Most of the time we have to sent the mail on the registered User's Email address when User forgets its password. Django send's the mail consisting of a randomly generated token which can be used for one time to reset user's password. But now lets not go further into token generation, lets discuss about how to configure the server to send mail to the User's Email address.

SMTP(Simple Mail Transfer Protocol) is the server that is used to send mail. For setting this server we have to configure it in our settings files. Here is the piece of code that I implemented in my settings/base.py file for SMTP server :-

First of all set the EMAIL_BECKEND variable in settings to "django.core.mail.beckends.smtp.EmailBackend"


#configuring SMTP
EMAIL_USE_TLS = True
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_HOST_USER = ''
EMAIL_HOST_PASSWORD = ''
DEFAULT_FROM_EMAIL = ''
DEFAULT_TO_EMAIL = 'to email'

Set EMAIL_HOST_USER,PASSWORD to the username and password of the email from which email has to be sent to users. Set the DEFAULT_FROM_MAIL to the email id from which mail has to be sent.

Import send_mail from django.core.mail in views.py. When you set all these configuration then its time to send mail. :P.

How it works?

I am not going into the deep explanation of working but in a nutshell we can say when we send an email, our e-mail client interacts with the SMTP server to handle the sending. Now SMTP will serve the email to the client email server and again through the client email server our client will receive the mail.

That's all regarding SMTP settings,configuration and a little bit of working.

Cheers!!

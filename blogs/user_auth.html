User Authentication in Django

How to register new user and how to authentiicate new user on login.

Posted by Prakhar Joshi on June 12, 2014

User authentication is easy to handle in Django. Django comes with an inbuilt authentication system which is really efficient and time saving. The basic thing which is required for authentication system is the registration of users and then login for the registered users only. So first of all we need some space(in terms of database, we need tables) to register users.

For that we will import the inbuilt User model from 'django.contrib.auth.models' (in models.py) to create tables in database. After importing these things we just need to sync the db and tables have been created in database. That's the power of Django.

Now after db creation we just need to work on the user forms from which data will be stored in tables . That is just the regular Django stuff,nothing new.

But when we make login form, here we again have to use the power of django. The process that occures during login is quite simple. The user enters the Username and password to login with. With the help of authetication system Django checks that username in the database and then authenticate the user with successful login if the username and password matches in db.

That was more or less verbal talks. But as always implementation is quite arduous. So here is the piece of code that authenticate the registered users through login form

:- from django.contrib.auth import authenticate
def login():
    if request.method == "POST":
        username = request.POST['username']
        password = request.POST['password']
        user = auth.authenticate(username = username, password = password)
    if user is not None:
        auth.login(request,user)
        return HttpResponseRedirect('/sucesspage')
    else:
        return HttpResponseRedirect('/invalid')
    return render(request,loginform.html)
This function will authenticate only the registered users only. Hope this will help you.
